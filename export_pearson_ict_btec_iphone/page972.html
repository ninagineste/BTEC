Remember

Modifying a table design has the greatest potential for causing additional work, as you may need to revisit and correct all of the relationships, queries and reports that depend on that table. It’s very important to get it right first time!

Did you know?

As in other software packages, formatting a field will only change the way the data is displayed. It does not change the actual stored values themselves. For example, some text may be actually stored in lower case but you want to display it as UPPERCASE text. Another example might be where you want negative numeric values to be displayed in red to highlight them in a field and this can also be achieved through formatting.

Did you know?

  • You can set field sizes to match the expected data. For text fields you can enter the maximum expected number of characters of the longest entry. For numeric fields you can select from a wide range of number types as well as from ‘number’. These include byte, integer, long integer, single, double and decimal.
  • A Default value is one inserted into the field by the system as the record is created unless the user enters a new value themselves. If a sales record has a ‘date sale took place’ field, you might choose a default value of ‘Today’ because that will be the appropriate value for most of the entries.
